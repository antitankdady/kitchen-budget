# プロジェクトルール

このプロジェクトでは「仕様駆動開発 (Specification Driven Development)」を採用します。

## 1. 開発プロセス (SDD)
1.  仕様の更新: コードを変更する前に、必ず `.agent/general.md` (仕様書) を確認・更新する。
2.  実装: 更新された仕様に基づいてコードを実装する。
3.  検証: 実装が仕様通りか確認する。

## 2. 言語設定
- ドキュメント: 日本語
- コミットメッセージ: 日本語 (推奨)
- コードコメント: 日本語 (英語のコメントは残さない)
- AI応答: 日本語
- 強調表示: ドキュメント内で `**` (強調) は使用しない。

## 3. コーディング規約
- TypeScript: Strictモード有効。`any` の使用は極力避ける。
- Vue: Composition API (`<script setup>`) を使用。
- Styling: `src/css` 配下のCSSファイルで管理。CSS変数 (`variables.css`) を活用し、ハードコードされた色は避ける。

## 4. デプロイ
- GitHub Pages: `main` ブランチへのプッシュで `deploy.yml` ワークフローが走り、自動デプロイされる。
- ビルド: `npm run build` で `dist` ディレクトリが生成される。クロスプラットフォーム対応のため、`cp` コマンドではなくNode.jsスクリプトを使用する。

## 5. その他
- データ永続化: 現在は `localStorage` を使用。将来的に Google Drive 等への対応を検討中（Stubあり）。
